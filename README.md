# xyOps FreeIPA Management

Плагин событий xyOps для управления пользователями и группами FreeIPA через JSON-RPC API. Версия 2.2.0 добавляет динамический выбор пользователей и групп через **Storage Bucket / Bucket Menu**.

## Возможности

- список и поиск пользователей;
- просмотр одного пользователя;
- создание пользователя с добавлением в выбранные группы;
- удаление пользователя с сохранением записи или окончательное удаление;
- восстановление сохранённого пользователя;
- отключение, включение и разблокировка учётной записи;
- список и поиск групп;
- добавление пользователя в несколько групп;
- удаление пользователя из нескольких групп;
- синхронизация пользователей и групп FreeIPA в bucket xyOps для выпадающих списков.

## Как работают динамические меню

В `xyops.json` включён Storage Bucket с фиксированным ID:

```text
bfreeipacache
```

Операция **Синхронизировать меню пользователей и групп** получает данные из FreeIPA и возвращает структуру:

```json
{
  "users": [
    { "id": "ivanov", "title": "Иван Иванов (ivanov)" }
  ],
  "enabled_users": [],
  "disabled_users": [],
  "preserved_users": [],
  "groups": [
    { "id": "developers", "title": "developers — Разработчики" }
  ],
  "metadata": {
    "updated_at": "2026-07-20T10:30:00.000Z",
    "truncated": false,
    "counts": {}
  }
}
```

Поля плагина используют эти пути как Bucket Menu:

- `users` — все обычные пользователи;
- `enabled_users` — пользователи, которых можно отключить;
- `disabled_users` — пользователи, которых можно включить;
- `preserved_users` — удалённые с сохранением пользователи;
- `groups` — группы FreeIPA с поддержкой multi-select.

Bucket хранит снимок каталога. После создания, удаления, восстановления, включения или отключения пользователя, а также после изменения членства в группах, выполните синхронизацию повторно.

## Требования

- xyOps / xySat версии `1.0.83` или новее;
- Node.js 18 или новее;
- `npm`, `npx` и `git` на выбранном рабочем узле xyOps;
- HTTPS-доступ с рабочего узла к FreeIPA;
- доверенный сертификат CA FreeIPA либо файл CA, обычно `/etc/ipa/ca.crt`.

Плагин не собирает телеметрию и обращается только к настроенному серверу FreeIPA.

## Секреты

Создайте Secret Vault и назначьте его плагину или событиям:

```text
IPA_USERNAME=xyops-provisioner
IPA_PASSWORD=change-me
```

Необязательная переменная:

```text
IPA_CA_CERT_PATH=/etc/ipa/ca.crt
```

Используйте делегированную техническую учётную запись. Не используйте встроенную учётную запись `admin` для регулярной автоматизации.

## Импорт в xyOps

1. Откройте **Plugins**.
2. Нажмите **Import File...**.
3. Загрузите `xyops.json` из корня репозитория.
4. Проверьте импортируемые объекты:
   - плагин `Управление FreeIPA`;
   - bucket `Кэш каталога FreeIPA` с ID `bfreeipacache`.
5. Подтвердите импорт.
6. Назначьте Secret Vault с `IPA_USERNAME` и `IPA_PASSWORD`.

Команда запуска в импортируемом файле:

```bash
npx -y github:askarahodov/xyops-freeipa-management-v2.1.0#main
```

## Первичное заполнение меню

Сразу после импорта меню пользователей и групп пустые, потому что XYPDF импортирует метаданные bucket, но не его данные.

### 1. Создайте событие синхронизации

Создайте Event:

```text
Title: Sync FreeIPA Directory Cache
Plugin: Управление FreeIPA
Operation: Синхронизировать меню пользователей и групп
Target: рабочий xySat с доступом к FreeIPA
```

Рекомендуемый `List limit`:

```text
10000
```

### 2. Добавьте действие Store Bucket

Добавьте к событию Action:

```text
Condition: On Success
Action: Store Bucket
Bucket: Кэш каталога FreeIPA
Sync: Data
```

Плагин выводит подготовленный кэш в `output.data`, а Store Bucket сохраняет его в `bfreeipacache`.

### 3. Запустите событие вручную

После успешного запуска проверьте bucket. В нём должны появиться ключи:

```text
users
enabled_users
disabled_users
preserved_users
groups
metadata
```

После этого поля выбора в операциях FreeIPA будут заполнены.

### 4. Добавьте расписание

Практичный вариант:

```text
каждые 5–15 минут
```

Также можно создать workflow:

```text
Изменение пользователя или групп
            |
            | success
            v
Sync FreeIPA Directory Cache
```

Так меню обновится сразу после операции.

## Выбор пользователей по операциям

| Операция | Источник меню |
|---|---|
| Показать пользователя | `users` |
| Удалить пользователя | `users` |
| Восстановить пользователя | `preserved_users` |
| Отключить пользователя | `enabled_users` |
| Включить пользователя | `disabled_users` |
| Разблокировать пользователя | `users` |
| Добавить/удалить из групп | `users` + `groups` |
| Создать пользователя | новый login вручную + `groups` |

## Примеры

### Создание пользователя

Выберите **Создать пользователя**:

```text
Логин: ivanov
Имя: Иван
Фамилия: Иванов
Email: ivanov@example.local
Группы: developers, vpn-users
Оболочка: /bin/bash
Домашний каталог: /home/ivanov
```

Группы выбираются через multi-select. Плагин получает массив, например:

```json
{
  "groups": ["developers", "vpn-users"]
}
```

### Добавление пользователя в группы

1. Выберите пользователя из списка.
2. Выберите одну или несколько групп.
3. Запустите job.
4. После успеха повторно синхронизируйте кэш.

### Удаление пользователя

Для удаления требуется флаг **Подтвердить удаление**.

Оставьте включённым **Сохранить удалённого пользователя**, чтобы запись появилась в меню восстановления после следующей синхронизации.

## Выходные данные

### Список пользователей

```json
{
  "freeipa_users": [],
  "freeipa_user_ids": [],
  "freeipa_users_truncated": false
}
```

### Список групп

```json
{
  "freeipa_groups": [],
  "freeipa_group_names": [],
  "freeipa_groups_truncated": false
}
```

### Изменяющие операции

Операции создания, удаления, восстановления, включения, отключения и изменения групп возвращают:

```json
{
  "freeipa_cache_refresh_required": true
}
```

Этот флаг можно использовать в workflow как признак необходимости повторной синхронизации каталога.

## Настройка команды репозитория

Скрипт позволяет изменить owner, имя репозитория и ref в `xyops.json`:

```bash
./configure-repository.sh <owner> [repository] [ref]
```

Пример:

```bash
./configure-repository.sh askarahodov xyops-freeipa-management-v2.1.0 main
```

## Безопасность

- держите проверку TLS включённой;
- храните пароль сервисной учётной записи только в Secret Vault;
- выдавайте сервисной учётной записи минимально необходимые права;
- ограничьте право редактировать и запускать административные события;
- учитывайте, что данные bucket являются кэшем и могут устареть между синхронизациями;
- FreeIPA остаётся источником истины: устаревший выбор будет повторно проверен сервером при выполнении операции;
- отдавайте предпочтение удалению с сохранением, пока процесс восстановления не протестирован.

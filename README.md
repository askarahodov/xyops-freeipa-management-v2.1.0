# xyOps FreeIPA Management

Набор Event Plugins xyOps для управления пользователями и группами FreeIPA через JSON-RPC API.

Версия `2.2.1` исправляет ошибку импорта:

```text
API Error: Toolset Data Error: Tool '...' field '...' type is invalid
```

## Почему возникала ошибка

xyOps поддерживает `Bucket Menu` как параметр плагина верхнего уровня, но внутри `toolset.fields` допускает только:

```text
checkbox, code, json, hidden, select, text, textarea
```

Поэтому динамические поля пользователей и групп перенесены на верхний уровень формы.

## Что импортируется

Файл `xyops.json` содержит:

1. `Кэш каталога FreeIPA` — Storage Bucket `bfreeipacache`;
2. `Управление FreeIPA` — основные операции;
3. `FreeIPA — Создать пользователя` — отдельная форма создания;
4. `FreeIPA — Восстановить пользователя` — отдельная форма восстановления.

Все три Event Plugins запускают одну реализацию:

```bash
npx -y github:askarahodov/xyops-freeipa-management-v2.1.0#main
```

## Поддерживаемые операции

Основной плагин:

- список пользователей;
- просмотр пользователя;
- удаление пользователя;
- отключение, включение и разблокировка;
- список групп;
- добавление пользователя в группы;
- удаление пользователя из групп;
- синхронизация кэша пользователей и групп.

Отдельные плагины:

- создание нового пользователя;
- восстановление сохранённого пользователя.

## Требования

- xyOps / xySat `1.0.83` или новее;
- Node.js 18+;
- `npm`, `npx` и `git` на рабочем xySat;
- HTTPS-доступ к FreeIPA;
- CA FreeIPA, обычно `/etc/ipa/ca.crt`.

## Secret Vault

Назначьте всем FreeIPA-событиям или плагинам:

```text
IPA_USERNAME=xyops-provisioner
IPA_PASSWORD=change-me
```

Необязательно:

```text
IPA_CA_CERT_PATH=/etc/ipa/ca.crt
```

Используйте делегированную сервисную учётную запись, а не встроенный `admin`.

## Импорт

1. Откройте **Plugins**.
2. Нажмите **Import File...**.
3. Импортируйте `xyops.json`.
4. Подтвердите создание bucket и трёх Event Plugins.
5. Назначьте Secret Vault.
6. Создайте событие первичной синхронизации.

## Первичная синхронизация меню

Создайте Event:

```text
Plugin: Управление FreeIPA
Operation: Синхронизировать меню
List limit: 10000
Target: xySat с доступом к FreeIPA
```

Добавьте действие:

```text
Condition: On Success
Action: Store Bucket
Bucket: Кэш каталога FreeIPA
Sync: Data
```

После запуска bucket должен содержать:

```text
users
enabled_users
disabled_users
preserved_users
groups
metadata
```

Рекомендуется запускать синхронизацию каждые 5–15 минут и после изменяющих операций.

## Основной интерфейс

В `Управление FreeIPA` динамические поля расположены над Toolset:

- **Пользователь** → `users`;
- **Группы** → `groups`, multi-select;
- **Фильтр по группе** → `groups`.

После заполнения общих полей выбирается операция.

### Добавить пользователя в группы

1. Выберите **Пользователь**.
2. Выберите одну или несколько **Групп**.
3. Выберите операцию **Добавить пользователя в группы**.
4. Запустите job.

### Включить пользователя

1. Выберите пользователя с подписью `— отключён`.
2. Выберите операцию **Включить пользователя**.
3. Запустите job.

## Создание пользователя

Используйте отдельный Event Plugin:

```text
FreeIPA — Создать пользователя
```

Форма содержит логин, имя, фамилию, email, начальный пароль и динамический multi-select групп.

## Восстановление пользователя

Используйте отдельный Event Plugin:

```text
FreeIPA — Восстановить пользователя
```

Поле пользователя загружается из `preserved_users`.

## Обновление команды репозитория

```bash
./configure-repository.sh <owner> [repository] [ref]
```

Пример:

```bash
./configure-repository.sh askarahodov xyops-freeipa-management-v2.1.0 main
```

Скрипт обновляет команду запуска во всех FreeIPA-плагинах внутри `xyops.json`.

## Безопасность

- не отключайте TLS-проверку в production;
- храните пароль только в Secret Vault;
- выдавайте сервисной учётной записи минимальные права;
- ограничьте право запуска административных событий;
- bucket является кэшем, а FreeIPA остаётся источником истины.

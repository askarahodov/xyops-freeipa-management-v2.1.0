# xyOps FreeIPA Management

Плагин событий для управления пользователями и группами FreeIPA через JSON-RPC API FreeIPA.

## Поддерживаемые операции

- список и поиск пользователей;
- просмотр одного пользователя;
- создание пользователя с необязательным добавлением в группы;
- удаление пользователя без возможности восстановления либо сохранение записи для последующего восстановления;
- восстановление сохранённого пользователя;
- отключение и включение учётной записи;
- разблокировка учётной записи после блокировки по политике паролей;
- список и поиск групп;
- добавление пользователя в группы;
- удаление пользователя из групп.

## Требования

- xyOps / xySat версии `1.0.83` или новее;
- Node.js 18 или новее;
- `npm`, `npx` и `git` на выбранном рабочем узле xyOps;
- доступ к FreeIPA по сети по протоколу HTTPS с рабочего узла;
- сертификат удостоверяющего центра (CA) FreeIPA, обычно `/etc/ipa/ca.crt`.

Плагин не собирает телеметрию и не отправляет данные никуда, кроме настроенного сервера FreeIPA.

## Секреты

Назначьте плагину Секретное хранилище xyOps (Secret Vault) со следующими переменными:

```text
IPA_USERNAME=xyops-provisioner
IPA_PASSWORD=change-me
```

Необязательно:

```text
IPA_CA_CERT_PATH=/etc/ipa/ca.crt
```

Используйте делегированную техническую учётную запись FreeIPA. Не используйте встроенную учётную запись `admin` для регулярной автоматизации.

## Импорт в xyOps

1. Откройте **Plugins** (Плагины).
2. Нажмите **Import File...** (Импортировать файл...).
3. Загрузите `xyops.json` из корня репозитория.
4. Проверьте импортированную команду и код, затем подтвердите импорт.
5. Создайте событие, используя **FreeIPA Management**.
6. Выберите рабочий узел xyOps, у которого есть доступ к FreeIPA.
7. Прикрепите Секретное хранилище (Secret Vault), содержащее `IPA_USERNAME` и `IPA_PASSWORD`.
8. Выберите операцию в поле **FreeIPA operation**, заполните поля и запустите событие.

## Поля подключения

- **FreeIPA URL**: `https://ipa.example.local`; `/ipa` добавляется автоматически.
- **FreeIPA CA certificate**: `/etc/ipa/ca.crt` на рабочем узле.
- **Request timeout**: по умолчанию `30` секунд.
- **Disable TLS verification**: только для тестирования; в производственной среде оставьте отключённым.

## Примеры операций

### Список пользователей

Выберите **List users** (Список пользователей). Необязательные значения:

```text
Search: ivanov
Only users in group: developers
List limit: 200
```

Результат отображается в виде таблицы и также возвращается как:

```json
{
  "freeipa_users": []
}
```

### Создание пользователя

Выберите **Create user** (Создать пользователя) и заполните поля:

```text
Login: ivanov
First name: Ivan
Last name: Ivanov
Email: ivanov@example.local
Groups: developers,vpn-users
Login shell: /bin/bash
Home directory: /home/ivanov
```

Начальный пароль необязателен. Назначение пароля администратором может привести к тому, что FreeIPA потребует смену пароля при первом входе, в зависимости от политики сервера.

### Отключение, включение и разблокировка

- **Disable user** (Отключить пользователя) вызывает `user_disable` и административно запрещает вход.
- **Enable user** (Включить пользователя) вызывает `user_enable`.
- **Unlock user** (Разблокировать пользователя) вызывает `user_unlock` и снимает временную блокировку, вызванную неудачными попытками аутентификации.

### Удаление пользователя

Для удаления требуется **Confirm deletion** (Подтвердить удаление).

Оставьте включённым **Preserve deleted user** (Сохранить удалённого пользователя), чтобы позже можно было выполнить восстановление через **Restore preserved user** (Восстановить сохранённого пользователя). Отключение сохранения выполняет окончательное удаление.

### Добавление пользователя в группы

Выберите **Add user to groups** (Добавить пользователя в группы):

```text
Login: ivanov
Groups: developers,vpn-users
```

Имена групп могут разделяться запятыми, точками с запятой или переводами строк.

## Примечания по безопасности

- Держите проверку TLS включённой.
- Храните пароль сервисной учётной записи только в Секретном хранилище xyOps (Secret Vault).
- Наделяйте сервисную учётную запись только теми правами, которые необходимы для используемых операций.
- Ограничьте круг лиц, имеющих право редактировать и запускать событие.
- Относитесь к полю начального пароля как к конфиденциальному, даже если оно скрыто в интерфейсе.
- Отдавайте предпочтение удалению с сохранением, пока не будет протестирован процесс восстановления.

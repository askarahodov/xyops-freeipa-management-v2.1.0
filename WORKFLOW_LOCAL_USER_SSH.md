# Workflow: локальный пользователь на SSH-хостах

Этот Event не использует FreeIPA. Он создаёт или обновляет одного локального Linux-пользователя на каждом адресе `host:port`.

## Импорт

Импортируйте строго по порядку:

1. `xyops-ssh-plugin.json` — должен установить Plugin `SSH — Создать локального пользователя`, ID `pmlc2ha8fssh_user`;
2. `workflow-local-user-ssh.json` — создаёт Event `Создать локального пользователя на SSH-хостах`.

## Secret Vault

Для подключения к серверам по закрытому ключу:

```text
SSH_USERNAME=automation-user
SSH_PRIVATE_KEY=-----BEGIN OPENSSH PRIVATE KEY-----...
SSH_PASSPHRASE=пароль_закрытого_ключа
```

Вместо ключа можно использовать:

```text
SSH_USERNAME=automation-user
SSH_PASSWORD=пароль_automation-user
```

Если административный SSH-пользователь выполняет sudo с паролем, добавьте:

```text
SSH_SUDO_PASSWORD=пароль_automation-user
```

`SSH_SUDO_PASSWORD` относится к административной SSH-учётке, а не к создаваемому пользователю.

## Поля запуска

- `Логин пользователя` — локальный Linux login;
- `Пароль пользователя` — задаётся через `chpasswd`;
- `Открытый SSH-ключ` — одна строка публичного ключа;
- `Разрешить sudo и sudo su` — добавляет пользователя в `sudo` или `wheel` и создаёт управляемое правило в `/etc/sudoers.d`;
- `Sudo без пароля` — создаёт правило `NOPASSWD`;
- `SSH-адреса и порты` — один `host:port` на строку.

Пример списка:

```text
192.168.32.15:5035
192.168.32.15:5037
server.example.local:22
```

## Что делает Job

На каждом хосте Job без интерактивных вопросов:

- создаёт пользователя или обновляет существующего локального пользователя;
- задаёт пароль;
- создаёт домашний каталог;
- создаёт `.ssh/authorized_keys`;
- записывает публичный ключ;
- опционально настраивает sudo;
- не меняет пароль root;
- не редактирует `sshd_config`;
- не устанавливает пакеты.

Если login уже существует только через LDAP/SSSD/NSS, локальный дубль не создаётся.

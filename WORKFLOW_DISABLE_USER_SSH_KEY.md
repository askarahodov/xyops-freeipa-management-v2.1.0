# Workflow: отключение FreeIPA-пользователя и удаление SSH-ключей

Workflow выполняет действия последовательно:

```text
Отключить пользователя FreeIPA
        ↓ только при успехе
Split списка SSH-хостов
        ↓
Удалить файл authorized_keys локального пользователя
```

## Импорт

1. `xyops.json`;
2. `xyops-ssh-plugin.json`;
3. `workflow-disable-user-ssh-key.json`.

## Поля запуска

- URL и CA-сертификат FreeIPA;
- логин пользователя;
- SSH-хосты по одному `host:port` на строку.

Публичный ключ указывать не нужно. SSH-шаг удаляет весь файл
`~/.ssh/authorized_keys` указанного локального пользователя.

Локальный пользователь, его пароль, домашний каталог и sudo не изменяются.
Если пользователя, каталога `.ssh` или файла `authorized_keys` на ноде нет,
SSH-шаг завершается успешно без изменений.

Удаление `authorized_keys` не завершает уже открытые SSH-сессии и не запрещает
вход по паролю, если он разрешён в конфигурации SSH-сервера.

# Workflow: локальный SSH-пользователь без FreeIPA

Workflow создаёт или обновляет локального Linux-пользователя на нескольких SSH-хостах без обращения к FreeIPA.

```text
Manual Trigger
      ↓
Split списка SSH-хостов
      ↓
SSH — Создать локального пользователя
```

## Импорт

1. Импортируйте `xyops-ssh-plugin.json`.
2. Убедитесь, что установлен Plugin `SSH — Создать локального пользователя` с ID `pmlc2ha8fssh_user`.
3. Импортируйте `workflow-local-user-ssh.json`.

## Поля запуска

- логин пользователя;
- пароль пользователя;
- открытый SSH-ключ;
- разрешение sudo;
- sudo без пароля;
- SSH-хосты по одному `host:port` на строку.

На каждой ноде создаётся или обновляется локальный пользователь, устанавливается пароль, создаются домашний каталог и `~/.ssh/authorized_keys`, после чего настраивается sudo.

Workflow не создаёт пользователя FreeIPA и не изменяет глобальный `sshd_config` или пароль root.

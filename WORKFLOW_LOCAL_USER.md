# Workflow: создание локального пользователя на SSH-хостах

Этот workflow не использует FreeIPA и не требует поля **Команда**.

## Импорт

Импортируйте файлы в таком порядке:

1. `xyops-ssh-plugin.json` — устанавливает Plugin `SSH — Создать локального пользователя`;
2. `workflow-local-user-ssh.json` — устанавливает Event `Создать локального пользователя на SSH-хостах`.

## Secret Vault

Назначьте Secret Vault плагину `SSH — Создать локального пользователя` или родительскому Workflow.

Для подключения по закрытому ключу:

```text
SSH_USERNAME
SSH_PRIVATE_KEY
SSH_PASSPHRASE
```

Если техническому SSH-пользователю для sudo нужен пароль:

```text
SSH_SUDO_PASSWORD
```

Если у технического пользователя настроен `NOPASSWD`, `SSH_SUDO_PASSWORD` не требуется.

## Поля запуска

Пользователь заполняет только:

- **Логин пользователя**;
- **Пароль пользователя**;
- **Открытый SSH-ключ пользователя**;
- **Разрешить sudo и sudo su**;
- **Sudo без пароля**;
- **SSH-адреса и порты** — по одному `host:port` на строку.

Пример списка хостов:

```text
192.168.32.15:5035
192.168.32.15:5037
server01.example.local:22
```

## Выполнение

Workflow содержит три элемента:

```text
Manual Trigger
      ↓
Split SSH hosts
      ↓
SSH — Создать локального пользователя
```

Для каждого хоста Plugin:

- создаёт или обновляет локального Linux-пользователя;
- задаёт пароль;
- создаёт домашний каталог;
- добавляет публичный ключ в `.ssh/authorized_keys`;
- опционально создаёт правило `/etc/sudoers.d/90-xyops-<username>`.

Workflow не изменяет пароль root, глобальный `sshd_config` и настройки FreeIPA.
